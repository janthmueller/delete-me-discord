---
title: First Run
description: Safest first-run workflow using discovery and dry-run mode.
---

The safest first run is deliberately narrow: discover IDs, choose one target, and preview the result in `--dry-run`.

This page assumes you already:

- installed the CLI
- authenticated with `dmd login` or `DISCORD_TOKEN`

## 1. Discover IDs first

```bash
dmd list guilds
dmd list channels
```

This gives you the IDs you need for `--include-ids` and helps avoid deleting from the wrong place.

At this stage, choose one specific target:

- one DM
- one Group DM
- or one guild text channel

Do not start with multiple IDs on your first run.

## 2. Start with dry-run

```bash
dmd clean --include-ids <channel_id> --dry-run
```

This example targets one channel. By default, it also removes your reactions and preserves nothing because:

- `--keep-last` defaults to `0`
- `--keep-within` defaults to `0`
- reactions are deleted unless you opt out with `--keep-reactions`

What you want to see here is:

- the correct channel name
- a believable number of messages or reactions in scope
- a per-channel summary that matches your expectation
- no surprise channels in the output

## 3. Execute only after the plan looks right

Drop `--dry-run` once you are satisfied with the output.

That gives you the same command shape, but with actual deletion:

```bash
dmd clean --include-ids <channel_id>
```

## How to read the dry-run output

Dry-run now reports per-channel planning details such as:

- messages and reactions that would be deleted
- `scan time`
- `est. execute time`
- `est. total time`

## Recommended progression

1. Start with one `--include-ids` target.
2. Use `--dry-run`.
3. Adjust `--keep-within`, `--keep-last`, and `--fetch-within`.
4. Only then expand scope or remove `--dry-run`.

Useful next adjustments:

- Add `--keep-within 2w` if you want to protect recent history.
- Add `--keep-last 20` if you always want to keep the last N messages in a channel.
- Add `--fetch-within 2w1d` later if repeated runs should only inspect a recent slice.

If you want a one-off cleanup of a single target, continue with [Full Channel Cleanup](../../guides/full-channel-cleanup/).

If you want repeated cleanup over time, continue with [Rolling Retention](../../guides/rolling-retention/).
