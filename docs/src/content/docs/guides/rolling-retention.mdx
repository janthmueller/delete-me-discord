---
title: Rolling Retention
description: Keep a recent window while expiring older messages over time.
---

Rolling retention is the right workflow if you want this tool to act more like maintenance than a one-time purge.

The goal is simple:

- keep a recent slice of history
- delete older content
- rerun the command regularly with predictable results

The key flags behind this workflow are:

- `--keep-within`
- `--keep-last`
- `--fetch-within`
- optionally `--preserve-cache`

## Time-only retention

Initial run:

```bash
dmd clean --keep-last 0 --keep-within "weeks=2" --dry-run
```

Recurring daily run:

```bash
dmd clean --keep-last 0 --keep-within "weeks=2" --fetch-within "weeks=2,days=1" --dry-run
```

## Time and count retention

Initial run:

```bash
dmd clean --keep-within "weeks=2" --keep-last 20 --preserve-cache --dry-run
```

Recurring daily run:

```bash
dmd clean --keep-within "weeks=2" --keep-last 20 --fetch-within "weeks=2,days=1" --preserve-cache --dry-run
```

## Guidance

For recurring runs, set `--fetch-within` slightly wider than your retention window so the tool can still see the transition boundary cleanly.

Examples:

- keep the last 2 weeks: use `--keep-within "weeks=2"`
- process only the recent slice on recurring runs: use `--fetch-within "weeks=2,days=1"`
- keep a minimum number of recent messages too: add `--keep-last`

If you also use `--keep-last`, the [Preserve Cache](../../reference/preserve-cache/) page explains why caching kept IDs helps repeated runs stay stable.

If you are unsure about the accepted time formats, see [Time Deltas](../../reference/time-deltas/).
