---
title: Rolling Retention
description: Keep a recent window while expiring older messages over time.
---

Rolling retention is the right workflow if you want this tool to act more like maintenance than a one-time purge.

The goal is simple:

- keep a recent slice of history
- delete older content
- rerun the command regularly with predictable results

The key flags behind this workflow are:

- [`--preserve-last`](../../reference/cli-options/#--preserve-last)
- [`--preserve-n`](../../reference/cli-options/#--preserve-n)
- [`--fetch-max-age`](../../reference/cli-options/#--fetch-max-age)
- optionally [`--preserve-cache`](../../reference/cli-options/#--preserve-cache)

## Time-only retention

Initial run:

```bash
delete-me-discord --preserve-n 0 --preserve-last "weeks=2" --dry-run
```

Recurring daily run:

```bash
delete-me-discord --preserve-n 0 --preserve-last "weeks=2" --fetch-max-age "weeks=2,days=1" --dry-run
```

## Time and count retention

Initial run:

```bash
delete-me-discord --preserve-last "weeks=2" --preserve-n 20 --preserve-cache --dry-run
```

Recurring daily run:

```bash
delete-me-discord --preserve-last "weeks=2" --preserve-n 20 --fetch-max-age "weeks=2,days=1" --preserve-cache --dry-run
```

## Guidance

For recurring runs, set `--fetch-max-age` slightly wider than your retention window so the tool can still see the transition boundary cleanly.

Examples:

- keep the last 2 weeks: use [`--preserve-last`](../../reference/cli-options/#--preserve-last) with `--preserve-last "weeks=2"`
- process only the recent slice on recurring runs: use [`--fetch-max-age`](../../reference/cli-options/#--fetch-max-age) with `--fetch-max-age "weeks=2,days=1"`
- keep a minimum number of recent messages too: add [`--preserve-n`](../../reference/cli-options/#--preserve-n)

If you also use `--preserve-n`, the [Preserve Cache](../../reference/preserve-cache/) page explains why caching preserved IDs helps repeated runs stay stable.

If you are unsure about the accepted time formats, see [Time Deltas](../../reference/time-deltas/).
