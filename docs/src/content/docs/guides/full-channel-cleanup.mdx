---
title: Full Channel Cleanup
description: Safely wipe one channel or DM with a dry-run first.
---

Use this workflow when you want to clean a single DM or channel as completely as possible.

## Recommended process

1. Discover IDs first.
2. Scope the run to one target with `--include-ids`.
3. Run `--dry-run`.
4. Execute only after the plan looks correct.

## Discover the target

```bash
dmd list guilds
dmd list channels
```

## Preview a full wipe

```bash
dmd clean --include-ids <channel_id> --dry-run
```

This means:

- only the selected target is processed
- no recent messages are kept unless you set `--keep-within`
- no "last N messages" are kept unless you set `--keep-last`
- your reactions are removed by default

## Execute the wipe

Drop `--dry-run` once you are satisfied:

```bash
dmd clean --include-ids <channel_id>
```

## Notes

- `--max-messages` can be useful if you want to test on a smaller slice first.
- If you want better per-channel timing visibility for a manual run, add `--buffer-per-channel`.
- If you only want a rolling cleanup instead of a full wipe, use [Rolling Retention](../rolling-retention/).
